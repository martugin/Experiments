	I. Создание хоста [MRHost]

1. Создаем класс-сервис
	1) Подключаем библиотеку System.ServiceModel
	2) Клепаем интерфейсы. Навешиваем на них [ServiceContract]
	3) На каждый метод (внутри интерфейса), который будем юзать, - [OperationContract]
2. Создаем непосредственно сам хост
	1) Подключаем библиотеку System.ServiceModel
	2) Формируем настройки хоста (mex, базовые адреса, конечные точки)


	II. Создание исполнимого клиента [MRClientExec]

1. Генерация прокси
	1) запустить хост
	2) Проект -> Добавить ссылку на службу. Ввести адрес запущенного хоста. Перейти.
	   Автоматически сгенерируется конфигурационный файл.
	3) Добавляем using {Подключенную службу}
	4) Хоть клиентский файл конфигурации собран, программно можем выбирать любую из
	   точек доступа хоста. Вообще, App.config можно полностью очистить: <configuration></configuration>
	   А еще лучше вообще удалить его =)
2. Реализация клиента

	III. Создание dll-клиента [MRClientLib]
1. Настраиваем свойства проекта
	1) Приложение -> Тип выходных данных - Библиотека классов
	2) Приложение -> Сведения о сборке -> Сделать сборку видимо для COM
	3) Подписывание -> Создать (без пароля лучше)
2. Генерация прокси (как в случае II)
3. Реализация клиента
4. Регистрация библиотеки
	1) в командной строке
		команда: C:\WINDOWS\Microsoft.NET\Framework\v4.0.30319\regasm (В корне тут тоже)
		параметры к ней: [путь к библиотеке]{name}.dll /tlb {name}.tlb /codebase {name}

	IV. Создание БД
вместо New стоит использовать CreateObject("{Lib name}.{class name}")
